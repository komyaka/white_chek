# 📋 Информация о парсере

С полного нуля мной был создан софт, который автоматизирует поиск и отбор конфигураций. 

Опишу его полную работу:

# [Поиск и обработка]

1. Софт ищет по ключевым словам файлы с конфигами VLESS всех типов. (на данный момент нет)

2. Затем найденные источники сохраняет в файл sources.txt (прикреплён к репо), чтобы не повторять поиск уже найденных источников.

3. Из сохранённых источников происходит скачивание конфигов.

# [Чистка]

4. Производится удаление дубликатов, битых и невалидных конфигураций, чтобы не нарушить дальнейшую работу софта.

# [Фильтрация конфигов]

5. Очищенные конфиги проходят отбор по iP и SNI, и софт оставляет только те конфигурации, которые возможно заработают при белых списках.

Список iP-адресов: https://raw.githubusercontent.com/RKPchannel/RKP_bypass_configs/refs/heads/main/iP.txt

Список доменов: https://raw.githubusercontent.com/RKPchannel/RKP_bypass_configs/refs/heads/main/domain.txt

# [Переименование конфигов]

6. После фильтра производится переименовывание конфигов, чтобы сразу видеть какого типа конфиг, через что проходит трафик, и в конце поставил [#РКП]

7. Конфиги перекодирутся для дальнейшего импортирования в клиенты. Некоторые VPN не поддерживают не захэшированное название конфига. Этот шаг исключает данную проблему.

# [Проверка конфигураций]

8. Готовенькие конфиги проходят URL-Проверку через ядро Xray по домену via PROXY, как у большинства клиентов, например v2RayNG. Это необходимо, чтобы минимизировать количество нерабочих конфигов у вас.

## ✍️ Справка использования парсера.
1. Парсер создан ИИ, я не владею навыками знания языка Python. Если вы решите мне помочь, не удивляйтесь, что код писал индус.
2. Парсер выложен исключительно для развития и оптимизации.
3. Парсер был создан с опорой на работу со слабым железом.
4. URL-Проверка парсера заведомо неправильная. Причиной этому является незнание полной реализации проверки, а также в зависимости от вашего местоположения разное колиичество конфигов у вас может работать. Это зависит от вашего провайдера.
5. Очистка дубликатов конфигураций была заведомо реализована по всем настройкам конфигов, кроме порта. Причиной этому является душение РКНом порта 443, и один конфиг в сборнике может иметь помимо 443 порта например 8443 порт, и я считаю, что это не является дубликатом.
6. В итоговой версии парсера выпилен поиск источников по причине того, что все возможные источники уже найдены.
7. Фильтрация в итоговой версии реализована только по iP + SNI. Отсутствие CIDR заключается в неудачных моих попытках отбора. В основном все CIDR добавлялись с доменами, которые не будут работать при белых списках.
8. Количество найденных конфигов постоянно меняется, нет стабильного числа из-за обновлений конфигов в источниках.
9. Парсер был написан по советам подписчиков #РКП, спасибо вам большое.
10. Ваши версии доработок парсера с удовольствием приму, напишите мне в Telegram: @RKP_admin
11. Если есть замечания в работе парсера, напишите в чат, мы сможем решить проблемы: @RKP_group
